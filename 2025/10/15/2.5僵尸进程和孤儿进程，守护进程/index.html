2.5僵尸进程和孤儿进程，守护进程

发表于 2025-10-15 分类于 Blog
  • 我们知道在unix/linux中，正常情况下，子进程是通过父进程创建的，子进程在创建新的进程。
  • 子进程的结束和父进程的运行是一个异步过程,即父进程永远无法预测子进程 到底什么时候结束。 # start
  • 当一个 进程完成它的工作终止之后，它的父进程需要调用wait()或者waitpid()系统调用取得子进程的终止状态。 # 后面讲的 join

【一】僵尸进程

【1】什么是僵尸进程

  • 僵尸进程是指完成自己的任务之后，没有被父进程回收资源
  • 导致它仍然存在于进程列表中
  • 但已经被停止运行
  • 这些僵尸进程将占用一定的系统内存空间，并且会影响程序的性能

【2】解决办法（UNIX）

  • windows上无解决办法

【3】示例

  • 主线程启动，主线程结束
  • 主线程结束了，子线程还在执行，就叫僵尸进程

【二】孤儿进程

【1】什么是孤儿进程

  • 孤儿进程是指父进程在子进程杀死之前就已经终止了

  • 导致了进程失去了与父进程通信的能力

  • 这些子进程（孤儿）就会被init进程接管

  • 把所有的没人接管的子进程全部杀死

【2】例子

  • 子进程存活，父进程死亡

    • 没有父进程来帮忙回收pid号

    • 操作系统会开设一个儿童福利院(init 进程)专门管理孤儿进程回收相关资源

【三】僵尸进程和孤儿进程谁的危害性更大？

  • 僵尸进程的危害更大
    • 因为僵尸进程一直会持续存在在后台占用资源
  • 孤儿进程因为主进程死亡后，子进程会被init接管并销毁

【四】守护进程

【1】什么是守护进程

  • 在计算机开机时就已经运行的进程，并且一直在后台运行的一类特殊进程
    • 特殊在只要系统不关机，就会一直存在
  • 它们通常不与用户直接交互，也不接受标准输入和输出，而是在后台执行某种任务或提供某种服务
  • 守护进程是由管理员手动启动的，它们可以根据系统的启动而启动，一直在后台运行，知道程序关闭或停止
  • 常见的守护进程包括网络服务 (如 web 服务器、邮件服务器、 ftp 服务器等)、日志记录系统 (如系统日志服务、应用程序日志服务等) 等。
  • 守护进程通常在后台运行，不需要用户交互，并且有较高的权限，因此编写守护进程需要特别注意安全性和稳定性。

【2】主进程死亡，子进程未死亡

  • 加守护进程–》p.daemon = True
    • 加一个守护进程 ： 子进程随着主进程结束而结束
    • 守护进程加的位置一定要在 start 之前
    • 在启动之前为当前子线程添加额外的参数和限制
