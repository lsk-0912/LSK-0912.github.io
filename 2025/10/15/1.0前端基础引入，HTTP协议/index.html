前端基础引入

【一】前后端介绍

【1】什么是前端

  • 与用户进行交互，让用户输入数据以及展示响应数据的媒介

  • 就比如：浏览器界面，手机界面，电脑界面等

【2】什么是后端

  • 后端是对用户输入的数据进行校验和加密逻辑的位置

  • 用于真正执行业务逻辑的代码

  • 就比如：python代码，java代码等

【二】前端的学习思路

【1】前端三剑客

  • HTML----》骨架，没有任何样式
    • 有一个人没穿衣服
  • CSS-----》给骨架添加样式
    • 给那个人买了一身衣服
  • JavaScript（JS）-----》控制界面的动态效果
    • 穿好衣服的人动起来

（1）HTML介绍

  • HTML指的是超文本语言，它是用来描述网页的一种语言
  • HTML不是变成语言，而是一种标记语言
  • 标记语言是一套标记标签

（2）CSS介绍

  • 层叠样式表，是用来为结构化文档（HTML、XML等应用）添加样式,比如字体、颜色、大小、间距的计算机语言

（3）javascript介绍

  • JavaScript是一种运行在客户端 的脚本语言

【2】Bootstrap(HTML+CSS)

  • 做前后端项目结合的时候，因为Bootstrap 小而轻便并且好上手
  • 在后面开发过程中如果对Vue掌握不牢直接用Bootstrap

【三】前端框架

  • BOOTSTRAP/JQuery/Vue
  • Bootstrap: 轻便前端框架
  • 提前封装好了很多操作，只需要按照固定的语法调用即可

浏览器访问地址全过程

【一】在浏览器中输入网址（URL）

【1】用户在浏览器的地址栏中输入目标网址www.biadu.com/

【2】www.biadu.com/ ：叫域名

  • 域名是用串英文字符来代表 一串ip和port的标志

  • 有了域名需要做DNS解析，得到ip和port

  • 域名就会不断地找服务器做域名直到解析出指定的额ip和port

【3】让ip和port和服务器建立连接 TCP连接

  • 三次握手
    • 第一次：客户端向服务端提交请求
    • 第二次：服务端回复请求
    • 第三次：建立联系

【4】TCP的连接

  • 流式协议：数据就会一直传输，不断的向你的服务器发送请求和携带响应的数据

【5】后端服务器接收到请求和携带的数据

  • 将数据进行处理
  • python的后端框架：Django / flask /fastapi

【6】数据处理完后返回给客户端指定的响应内容

  • 在响应内容中会携带相应的相应标志

【7】前端拿到响应数据

  • 在页面渲染指定的结果或者页面

【8】渲染完成后和服务端断开连接

  • 断开的连接不是指TCP的连接

  • 指的是本次的请求断开

HTTP协议初始

【一】那些应用可以充当客户端？？？

  • 自己写的TCP客户端
  • 我们用的浏览器
  • cs架构里面符合c的程序
  • bs架构里面符合b的程序

【二】我们自己写的TCP的客户端无法和浏览器进行交互的原因

  • 浏览器和服务端之间遵循HTTP协议

【三】什么是HTTP协议

  • 规定了服务端和浏览器之间数据交互的格式
  • 如果服务端遵循了HTTP协议，就可以被浏览器识别到
  • 如果服务端不遵循HTTP协议 浏览器就不会正常访问和展示 但是不影响服务端
  • 如果应用非常火爆，可以做成app客户端

HTTP协议

【一】什么是HTTP协议

  • HTTP协议（HyperText Transfer Protocol，超文本传输协议）

    • 由蒂姆·伯纳斯-李于1989年为了推广互联网技术而推出的一种无状态网络应用协议

  • HTTP协议构建于TCP/IP协议族之上

    • 属于应用层协议

  • 主要用于传输与超文本相关的资源文件

    • 如HTML网页，css，is，图片，视频，音频等

  • 下载的HTTP协议基于数据是否加密，主要分两种

    1. 一种是正常的HTTP协议
      • 没有进行加密传输数据的就是HTTP协议
      • 默认端口为80端口
      • 常用端口3000,5000,8000,8080等
    2. 一种是TLS/SSL认证的HTTPS协议
      • 构建与TLS或SSL协议之上
      • 对数据使用SSL加密传输数据的是HTTPS协议
      • 默认端口是443端口
      • 常用端口：8443

HTTP协议的特点----》重点

【一】特点

【1】基于请求和响应的协议

  • 服务端永远不会主动联系客户端
  • 所有数据都是有客户端发起
  • 服务端必须做出响应

【2】HTTP协议是基于TCP/IP协议之上的应用层协议

  • 应用层协议:HTTP HTTPS FTP …

【3】无状态

  • 服务端不保存客户端状态（不管见了多少遍，每次见面都相当于是初见）

  • 无状态指的是http协议默认的情况下，服务端不识别客户端

  • 在客户端多次发起请求到同一个服务器，服务端接收到客户端的请求在处理完成以后就会主动断开。

  • 对于客户端的每次请求，对于服务端来说，都是一次新的客户端请求

  • 服务端无法辨别多次请求的客户端是否是同一个客户端

【4】短连接

  • 不保持客户端之间的连接通道
  • 服务端响应数据，立马就会断开连接

【5】长连接–》websocket

  • 客户端和服务端不断地在数据交互直至数据拿到并处理，处理完成之后不会断开连接

  • 主要用于加好友聊天业务

HTTP协议的请求和响应的格式

【一】HTTP协议的请求格式

（1）客户端给客户端发送消息应该遵循的数据格式

  1. 请求首行
    • 请求方式 + 请求路径 + 协议版本
  2. 请求头
    • 一大堆k:v键值对
  3. 换行不能省略—》/r/n
  4. 请求体(携带敏感数据:密码 身份照号…) 不是一直都有

（2）HTTP请求头（Request Head）

  • 主要对客户端请求操作进行限制条件与补充说明

  • 常见的标准HTTP协议如下

选项 描述 值
Host 指定客户端请求的服务器的域名和端口号。 www.baidu.com
Content-Type 告诉服务器，客户端请求携带的请求体数据的媒体类型信息（MIME类型）
User-Agent 告知服务器HTTP 客户端网络代理程序的版本信息，一般就是浏览器的版本信息。
Authorization 告知服务器客户端的Web认证信息。
Content-Length HTTP报文中请求体的大小，以字节为单位。
Referer 告诉服务器该网页是从哪个页面链接过来。也就是上一页页面的地址。
Accept 指定客户端能够接收并理解的媒体类型类型（MIME类型），用于表达希望服务端的返回资源格式。
Accept-Encoding 指定浏览器可以支持的web服务器返回内容压缩编码类型。 gzip, deflate, br
Pragma 指定服务端控制缓存行为。http/1.0以前的字段。 Pragme: no-cache
Cache-Control 指定服务端控制缓存行为。http/1.1以后的字段。 Cache-Control: no-cache
Upgrade 向服务器请求在当前http协议的基础上升级采用新的某种传输协议以便服务器进行转换 常用于http协议升级到ws协议。
Connection 指定本次http通信结束以后，是否关闭TCP网络连接。如果设置持久连接，则可以在一次会话中，可以使用同一个TCP连接，进行多次的HTTP通信，提高效率。 持久连接，Connection: keep-alive 非持久连接，Connection: close

（4）请求方法

  • HTTP请求方法（HTTP Request Method）
  • 表示客户端希望对服务器指定资源进行哪一类型的操作
  • 存在多种HTTP请求方法表达增删查改

常见的请求方法如下：

请求方法 描述 报文中是否包含请求体
GET 表示客户端希望从服务器中获取或下载资源信息 Flase
POST 表示客户端希望上传文件或通过请求在服务器创建资源信息。 True
PUT 表示客户端希望修改或更新服务器资源（表示修改全部资源信息，例如数据表的一整个记录） True
PATCH 表示客户端希望修改或更新服务器资源（表示修改部分资源信息，例如数据表的一个记录里面某个属性值） True
DELETE 表示客户端希望删除或废弃服务器资源 Flase
OPTION 表示客户端希望获取服务器所支持的请求方法列表 Flase
HEAD 表示客户端希望获取服务器支持的跨域地址列表 Flase

（5）请求行

  • 请求行（HTTP Request Line）
    • 表示请求报文的首行
  • 主要有三部分组成，使用单个空格隔开
    • 请求方法
    • 请求路径
    • HTTP协议版本

（6）请求路径

  • 表示远程web服务端的一个可访问资源
  • 一般就是代表的就是一个服务器的具体文件或数据表中的记录信息，或一个服务端里面的函数或方法
