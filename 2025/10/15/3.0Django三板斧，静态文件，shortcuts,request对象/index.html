Django三板斧

  • 三板斧都是从
  • 返回http响应，渲染HTML和重定向

【一】HttpResponse

  • HttpResponse：是django自带的类，由于构建基本的HTTP响应
  • 例子,在django的view.py文件下使用

  • 字符串渲染到页面上还是字符串
  • 字典格式的数据，渲染到页面的是字典所有的键
  • 前提是已经在urls.py里面渲染了这个函数

【二】render

  • 前端模板文件一点要放在app文件夹下的templates目录下

  • 参数介绍，render(request --> 必须传的参数, “index.html”—HTML文件名，locals()–> 局部名称空间)

    • 要想的到字典里面的数据，利用jinjia2方法，HTML界面通过 模板语法{{ }} 使用变量
    • locals()：可以写，不写不会执行函数里面的代码块，直接运行的就是HTML文件

【三】redirect

  • 主要用重定向，括号里可以写其他网站的全程，可以写自己网站的后缀
  • redirect：重定向路由请求地址 在状态码里面有是3xx状态码

【补充】

  • 只是对于前端来说

  • django自带了重启功能，ctrl + s 保存，保存以后不需要重启django直接刷新页面即可

  • 静态文件就是我们在开发过程中基本上不会更改的文件

【2】Django静态文件结构树

  • 静态文件夹都叫–>static
  • 对static文件夹进行划分
    • js文件夹
    • css文件夹
    • img文件夹
    • plugins文件夹

    • 如：<img src="../static/pic10.jpg" alt="...">

【三】在Django中使用静态文件

【1】在前端页面使用jQuery文件

  • cdn链接引入
  • 引入本地的文件不能用
    • 发现页面上的js时间不生效
    • 打开检查，网络 404

【2】静态文件加载

  • 方式一

    • 前提是你已经在settings.py文件中配置好了路径,有了静态文件目录你才能够用

    • STATIC_URL = '/static/'
STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'user/static')

  • 方式二

    • 前提是你已经在settings.py文件中配置好了路径,有了静态文件目录你才能够用

    • STATIC_URL = '/static/'
STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'user/static')

【2】你确认你有一个static文件夹

  • 将static放在APP文件目录下
  • Django项目在加载环境变量的时候，因为你已经注册了APP，所以你就能找到APP目录下的static文件夹
  • 如果放在根目录同级的目录下，会发生错误，因为Django的自身权限原因，加载不到除了当前APP以外的区域
  • 放在APP里面正常用，以外就不可以用

创建一的参数项-----以外需要在配置环境配置

【4】内部的原理

  • 你配置了STATIC_URL

  • 可以找到以http://127.0.0.1:8899/static/ ...

  • 访问静态文件的时候可以挨个检测资源目录 默认的资源目录就是APP下的目录

  • 将当前资源自动转换为指定的地址发出请求http://127.0.0.1:8899/static/ ...

【5】将当前APP的view源码文件放在了static目录下

  • 不可以，会导致数据泄露，你会赔死
  • 这样的话黑客随时可以看你的后端代码
