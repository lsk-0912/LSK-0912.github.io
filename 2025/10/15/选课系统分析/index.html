选课系统分析

``` 【一】功能分析 选课系统 管理员 ---> 负责管理后台 添加和管理信息 学生端 ---> 负责选课，上课 ... 讲师端 ---> 负责讲课 课程 ---> 有哪些课程 学校 ---> 在哪开设课程 【二】管理员模块 【1】注册 【2】登录 【3】添加学校 【4】添加讲师 【5】添加课程 【6】修改讲师密码 【三】讲师端 【1】登录 【2】讲师可以选择课程 【3】查看当前课程下的学生 【4】查看当前课程下的学生的分数 【四】学生端 【1】注册 【2】登录 【3】查看所有的课程信息 【4】选择课程 【5】查看课程分数 【五】管理员信息 管理员账号 管理员密码 【六】学校信息 学校名字 学校地址 学校简介 学校关联课程(学校允许开设多门课程) 【七】讲师 讲师账号 讲师密码 选择的课程信息 (讲师讲多门课) 【八】学生 学生姓名 学生账号 学生密码 选择的课程信息 (学生可以选择多门课程) 【九】课程 课程名称 课程周期 课程价格 关联学校 (一个课程只能有一个学校) 【十】架构设计 遵循三层架构原则 继承ATM 的分层架构风格 bin # 主文件层 -- main.py core # 视图展示层 -- admin.py -- teacher.py -- student.py conf # 配置文件层 -- settings.py interface # 逻辑接口层 -- admin_interface.py -- teacher_interface.py -- student_interface.py db # 数据存储和处理层 -- data # 负责存储所有的数据 -- Admin -- Student -- Teacher -- Course -- School -- db_handler.py 负责数据处理读取和写入 -- models.py 模型层 lib # 公共功能 -- common.py 存储公共的功能函数 log # 日志存储层 start.py # 启动入口 README.md # 项目解释说明书 【十一】架构选型 Python 面向对象和函数 魔法方法 + 绑定方法 + 非绑定方法 + 反射 数据库存储 ---> pickle pickle ---> 允许保存 python 的对象 不能实时更新，只能保存保存的那一时刻的数据 ```
