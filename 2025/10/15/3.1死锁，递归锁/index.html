3.1死锁，递归锁

发表于 2025-10-15 分类于 Blog

死锁

【一】死锁介绍

  • 死锁是指两个或多个进程，在执行过程中，因争夺资源而造成了互相等待的一种现象

  • 既两个或多个进程持有各自的锁并试图获取对方持有的锁，从而导致被堵塞，不能往下执行，最后形成僵局

  • 这种情况资源利用率极低，处于一种死循环的状态

  • 死锁就相当于：一个人有筷子，一个人有碗，都试图要对方的东西，就僵持在哪里，都吃不上饭

【二】解决方法

  • 锁不要有多，一个足够
  • 如果真的发生了思索的现象，必须迫使一方交出锁

【三】案例

  • 有两个锁，继承Thread方法
```python from threading import Thread, Lock import time metexA = Lock() metexB = Lock() # 类只要加括号多次 产生的肯定不同的对象 # 如果你想要实现多次加括号等到的是相同的对象 - 单例模式 class MyThread(Thread): def run(self): # 第一步：启动func1----》运行完 self.func1() # 第二步 self.func2() def func1(self): # 加锁 metexA.acquire() # self.name：获取当前线程名 # 打印锁A print(f'{self.name} 抢到了A锁') # 加锁B metexB.acquire() # 打印锁B print(f'{self.name} 抢到了B锁') # 解锁B metexB.release() # 解锁A metexA.release() def func2(self): # 加锁B metexB.acquire() # self.name：获取当前线程名 # 打印锁B print(f'{self.name} 抢到了B锁') # 睡了2s，在这期间就会继续继续回去，回到run里面继续执行func1 time.sleep(2) metexA.acquire() print(f'{self.name} 抢到了A锁') metexA.release() metexB.release() def main(): for i in range(10): t = MyThread() t.start() if __name__ == '__main__': main() # 输出 # Thread-1 抢到了A锁 # Thread-1 抢到了B锁 # Thread-1 抢到了A锁 # Thread-2 抢到了A锁 # 线程卡死 # 第一个线程正常执行完，执行第二个线程的时候，睡了两秒， # 这个时间就会继续回到run里面继续执行func1，但是第一次执行第二个线程的时候锁B还在睡，还在执行，就卡在那里了
