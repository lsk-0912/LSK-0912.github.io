发表于 2025-10-15 分类于 Blog

session对象

  • session：其实就是保持会话的状态
  • 我们用过
    • auth.login(request,user_obj),保存状态就是session来做的

【一】使用

【1】不使用session保持会话 直接使用Cookie

```python from fake_useragent import UserAgent import requests # 爬数据的请求URL target_url = "https://stock.xueqiu.com/v5/stock/batch/quote.json?symbol=SH000001,SZ399001,SZ399006,SH000688,SH000016,SH000300,BJ899050,HKHSI,HKHSCEI,HKHSTECH,.DJI,.IXIC,.INX" headers = { "User-Agent": UserAgent().random, "Cookie":"开发者模式，网络的第一个文件夹，把Cookie复制出来" } response = requests.get(url=target_url, headers=headers) page_text = response.text # 加上cookie就可以得到信息 print(page_text)

【3】使用session保持会话

```python import requests from fake_useragent import UserAgent # 生程一个会话对象 session = requests.Session() # 目标网址 headers = { "User-Agent": UserAgent().random } target_url = "https://xueqiu.com/" response = session.get(target_url, headers=headers) response.encoding = "utf8" # 你想要的到数据的URL target_url = 'https://stock.xueqiu.com/v5/stock/batch/quote.json?symbol=SH000001,SZ399001,SZ399006,SH000688,SH000016,SH000300,BJ899050,HKHSI,HKHSCEI,HKHSTECH,.DJI,.IXIC,.INX' response = session.get( url=target_url, headers=headers ) page_text = response.text print(page_text) ```

【4】小结

  • 解析

    • 第一次请求

      • 请求目标地址，获取不到数据，提示未登录

      • 于是复制Cookie携带Cookie

    • 第二次请求

      • 先用requests请求雪球网—>获取到cookie
      • 将cookie提取出来—》给请求使用
      • 请求成功

    • 第三次请求

      • 使用session对象保持会话状态
      • 生成session对象发起请求Cookie会自动加载到session对象中
      • 下一次请求直接使用session
