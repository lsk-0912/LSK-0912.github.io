【一】引入

  • 前端展示的数据有时候会有很多
  • 一页展示了很多数据，数据量很大，查找不方便

解决办法

  • 做分页处理

【二】分页推到

【1】分析

  • get请求是可以携带参数的
  • 由此我们可以在前端点击页数的时候，把这个页数一起发送给后端
  • queryset对象是支持索引取值和切片操作的
    • 不支持负索引取值

【2】代码分析

  • 后端代码分析
    • 在后端通过model.objects.all()得到所有数据的quertset对象
    • queryset对象可以进行切分，但是不支持反向切分
  • 前端展示数据
    • 在前端需传入当前的页面
    • 在后端根据指定的页面进行切分
    • 在前端的url上声明当前的页码
    • http://127.0.0.1:8000/?page=1
    • 后端的request.GET.get(‘page’)就可以拿到当前的页码
  • 在后端处理
    • 我们需要
    • 当前的页码curreny_page
    • 每一页展示的数据量size
    • 起始的页码 start_page
    • 结束的页码 end_page
    • queryset对象[起始位置：结束位置]
      • 列入 book_queryset = models.Book.objects.all()[start_page:end_page]

from user.paginations import Pagination

def index(request):
# 拿到模型表里面的所有数据
user_list = UserInfo.objects.all()
# 从路径里面拿到页面,默认第一页
current_page = request.GET.get(‘page’, 1)
# 拿到总页数
all_count = user_list.count()
# current_page 得到的页码数，all_count总页数，per_page_num每个页面只能展示多少条数据
page_obj = Pagination(current_page =current_page, all_count=all_count, per_page_num=10)
# 对象进行切分
page_queryset = user_list[page_obj.start:page_obj.end]
return render(request, ‘index.html’, locals())
