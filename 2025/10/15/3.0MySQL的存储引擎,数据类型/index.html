MySQL的存储引擎

【一】什么是存储引擎

  • 存储引擎就是不同的处理机制–》不同的数据应该有不同的处理机制
  • 存储引擎就好比–》针对不同的文件格式会有不同的存储方式和处理机制

【二】MySQL主要的引擎介绍

  • 四个主要的引擎介绍

【1】Innodb

  • 是MySQL5.5版本之后的默认存储引擎
  • 支持事务/行锁/外键

【2】myisam

  • 是MySQL5.5版本之前的默认存储引擎
  • 速度比innodb快，但数据安全较弱

【3】memory

  • 内存引擎
  • 数据全部放在内存中
  • 储存速度快，不能长时间保存，断电数据就会消失

【4】blackhole

  • 黑洞引擎
  • 无论存储多少数据都会立刻消失

【5】查看所有的数据库引擎

  • engines : 引擎

  • 控制MySQL里面存储数据的一些限制规则

  • 临时修改–》只在本窗口有效

  • 永久修改
  • 退出客户端重新连接，修改后的严格模式会一直生效
  • 修改完成后，重启服务端，即可生效

  • 严格模式在5.7版本之后都是默认开启严格模式的
  • 尽量减轻数据库的负担，不给数据库增加格外的压力

【4】char与varchar的比较

（1）char

（2）varchar

【六】总结

【1】整数类型

  • 用于储存整数值
  • BIGINT：大整数类型，可以存储更大范围的整数值

【2】浮点数

  • 高精度浮点数，用于储存更精确的小数数值

【3】日期类型

  • date : 存储年份

  • time ：存储时分秒

  • datetime : 存储年月日时分秒

  • timestamp : 存储年月日时分秒，并自动记录修改的时间戳

  • year ： 年

【4】字符长类型

  • 开启严格模式

  • char：自动补全长度

  • varchar：不会动不全

【5】枚举和集合

  • 枚举
    • 只能选择定义好的数据，而且只能选择一个，负责报错
  • 集合
    • 只能选择定义好的数据，否则会报错，可以多选，多选之间用逗号隔开

【6】约束条件

  • unsigned 无符号

    • unsigned约束条件用于指定某个字段只能包含非负数（正整数或零）。
    • 当将字段定义为unsigned时，它会限制该字段的取值范围为大于等于零的整数。
    • 这对于表示计数器、年龄、数量等非负数值的字段非常有用。

  • null 可以为空

  • not null 不可以为空

  • zerofill 某个字段的数据以零填充

    • zerofill约束条件用于指定某个字段的数据以零填充。

    • 当使用zerofill约束时，如果字段被赋予较短的值，MySQL将使用来填充该字段的剩余空间。

      • 例如，如果一个字段定义为INT(5) ZEROFILL
      • 并且赋值为123
      • 则存储在数据库中的值将是00123。
      • 这样做可以确保字段的固定长度，方便进行比较和处理。

【7】严格模式

  • 查看所有的严格模式
    • show variables like ‘%mode’;
    • %模糊查询任意长度
    • _模糊查询一个长度
  • 修改严格模式
    • 全局永久性修改
      • set global sql_mode = ‘’;
    • 局部临时性修改
      • set session sql_mode = ‘’;
