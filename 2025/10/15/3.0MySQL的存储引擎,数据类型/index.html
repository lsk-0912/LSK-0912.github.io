MySQL的存储引擎

【一】什么是存储引擎

  • 存储引擎就是不同的处理机制–》不同的数据应该有不同的处理机制
  • 存储引擎就好比–》针对不同的文件格式会有不同的存储方式和处理机制

【二】MySQL主要的引擎介绍

  • 四个主要的引擎介绍

【1】Innodb

  • 是MySQL5.5版本之后的默认存储引擎
  • 支持事务/行锁/外键

【2】myisam

  • 是MySQL5.5版本之前的默认存储引擎
  • 速度比innodb快，但数据安全较弱

【3】memory

  • 内存引擎
  • 数据全部放在内存中
  • 储存速度快，不能长时间保存，断电数据就会消失

【4】blackhole

  • 黑洞引擎
  • 无论存储多少数据都会立刻消失

【5】查看所有的数据库引擎

  • engines : 引擎
```sql show engines;

MySQL基本数据类型

【一】整数类型

  • 整型默认情况下都是带有符号的

  • 针对整型()内的符号到底是什么用？

    • 针对整型，括号内的宽度是用来限制存储数据的范围的

特例：只有整型括号内的数字不是表示限制位数而是显示长度

【1】分类

整数类型 字节 无符号数的取值范围 有符号数的取值范围
TINYINT 1 0~255 -128~127
SMALLINT 2 0~65535 -32768~32767
MEDIUMINT 3 0~16777215 -8388608~8388607
INT 4 0~4294967295 -2147483648~2147483647
BIGINT 8 0~18446744073709551615 -9223372036854774808~9223372036854774807

【2】作用

  • 用在储存的年龄，id，等级，电话等

【3】以TINYINT存储数据

  • 是否有符号
    • 默认情况下是带符号的
  • 超出最大范围，只存在最大接收值

（1）TINYINT存储数据

  • 超出最大储存数据
  • 创建表–>前提是在库里面
```sql create table two(id tinyint);

  • 插入数据，错误示范，超出数据TINYINT存储数据
```sql insert into two values(-129),(256); -- 报错 -- mysql> insert into two values(-129),(256); ERROR 1264 (22003): Out of range value for column 'id' at row 1

  • 默认情况下是带符号的
  • 超出限制只能保存最大接受值

（2）整数类型加约束条件（unsigned（无符号））

  • unsigned ：只能是正数和0
  • 创建表
```sql create table three(id tinyint unsigned);

  • 插入数据
```sql insert into t10 values(156);

  • 插入数据
    • 虽然没有报错但是还是要遵循最大范围
```sql mysql> insert into four values(1234567891);

（2）用zerofill

  • 创建表
```sql create table five(id int(8) unsigned zerofill);
  • 总共255位，小数部分占30位
```sql double(255,30)

【3】精准度验证

  • 创建表
```sql create table t15(id float(255,30)); create table t16(id double(255,30)); create table t17(id decimal(65,30));

```sql -- float : 从第八位开始错乱 -- double ：从第16位数据开始错乱 -- decimal 从第23数据开始错乱
  • VARCHAR
    • 边长
```sql varchar(4) --超过四个字符直接报错，不够四个字符,有几个就存几个

  • 多传超过四个字节就会报错

【3】补充char_length

char_length : 统计字段长度

```sql select char_length(name) from t18;

  • char 在硬盘上存的绝对是真正的数据，带有空格

【4】char与varchar的比较

（1）char

  • 缺点：浪费空间

  • 优点：存储简单

    • 直接按照固定的字符存储数据即可

（2）varchar

  • 优点：节省空间

  • 缺点：存储麻烦

    • 存的时候，在真正数据的前面加报头（表示数据真正大小）
    • 取的时候，需要先读取报头，才能读取真实的数据

以前用 char 现在 varchar 使用较多

【四】时间类型

【1】分类

  • date：年月日

  • datetime:年月日时分秒

  • time：时分秒

  • year：年

【2】示例

  • 创建表
```sql create table student( id int, name varchar(16), born_year year, birth date, study_time time, reg_time datetime );

【五】枚举和集合类型

【1】枚举和集合

（1）枚举（enum）：多选一

  • 枚举类型存储数据只能从候选项中选取一个

（2）集合（set）：多选多

  • 集合类型存储数据可从候选项中选取多个

【2】使用

（1）枚举

  • 创建表
```sql create table user(id int, name char(16), gender enum('male','female','other'));

(2)集合（set）

集合类型，可以多选，不可不选，选错就会报错

  • 创建表
```sql create table teacher( id int, name varchar(16), gender enum('male','female','others'), hobby set('read books','listen music','play games') );
  • 再插入数据
```sql insert into teacher values( 2, 'chimeng', 'female', 'read books,listen music' ); ```

【六】总结

【1】整数类型

  • 用于储存整数值
  • BIGINT：大整数类型，可以存储更大范围的整数值

【2】浮点数

  • 高精度浮点数，用于储存更精确的小数数值

【3】日期类型

  • date : 存储年份

  • time ：存储时分秒

  • datetime : 存储年月日时分秒

  • timestamp : 存储年月日时分秒，并自动记录修改的时间戳

  • year ： 年

【4】字符长类型

  • 开启严格模式

  • char：自动补全长度

  • varchar：不会动不全

【5】枚举和集合

  • 枚举
    • 只能选择定义好的数据，而且只能选择一个，负责报错
  • 集合
    • 只能选择定义好的数据，否则会报错，可以多选，多选之间用逗号隔开

【6】约束条件

  • unsigned 无符号

    • unsigned约束条件用于指定某个字段只能包含非负数（正整数或零）。
    • 当将字段定义为unsigned时，它会限制该字段的取值范围为大于等于零的整数。
    • 这对于表示计数器、年龄、数量等非负数值的字段非常有用。

  • null 可以为空

  • not null 不可以为空

  • zerofill 某个字段的数据以零填充

    • zerofill约束条件用于指定某个字段的数据以零填充。

    • 当使用zerofill约束时，如果字段被赋予较短的值，MySQL将使用来填充该字段的剩余空间。

      • 例如，如果一个字段定义为INT(5) ZEROFILL
      • 并且赋值为123
      • 则存储在数据库中的值将是00123。
      • 这样做可以确保字段的固定长度，方便进行比较和处理。

【7】严格模式

  • 查看所有的严格模式
    • show variables like ‘%mode’;
    • %模糊查询任意长度
    • _模糊查询一个长度
  • 修改严格模式
    • 全局永久性修改
      • set global sql_mode = ‘’;
    • 局部临时性修改
      • set session sql_mode = ‘’;
