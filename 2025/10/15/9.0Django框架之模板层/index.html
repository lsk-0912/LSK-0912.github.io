【2】{% %}：逻辑相关

  • {% %} 标签用于执行逻辑相关的操作，这些操作通常不会直接生成可见的内容，而是控制模板的渲染流程、加载其他模板标签、设置变量等

  • 逻辑相关的标签提供了在模板中进行条件判断、循环遍历、包含其他模板片段等功能

  • 条件判断 ({% if %}, {% elif %}, {% else %}, {% endif %}）

    • 用于在模板中进行条件判断，根据条件的结果渲染不同的内容。
  • 循环遍历 ({% for %}, {% empty %}, {% endfor %})
    • 用于遍历列表、元组、字典等可迭代对象，并在每次迭代中渲染模板内容。
  • 模板包含 ({% include %})
    • 用于在模板中包含另一个模板的内容。
  • 模板继承 ({% extends %}, {% block %}, {% endblock %})
    • 允许一个模板继承另一个模板的内容，并在需要的地方覆盖或添加内容。
  • 注释 ({% comment %}, {% endcomment %})
    • 用于在模板中添加注释，这些注释不会在渲染后的HTML中显示。
  • {% %} 标签中的逻辑操作是在服务器端进行的，而不是在客户端的浏览器中。
  • 当Django服务器接收到一个请求并决定使用某个模板来渲染响应时，它会解析模板中的 {% %} 标签，并根据标签中的逻辑执行相应的操作，然后将渲染后的HTML发送给客户端
