【2】render

  • render 函数是 Django 的模板渲染机制的一部分，通常与模板引擎（如 Django 的默认模板引擎 - Jinja2 或者其它支持的模板引擎）一起使用。
  • 当你想要返回带有动态内容的 HTML 页面时，可以使用 render 函数将请求的数据与预定义的 HTML 模板结合起来：
```python from django.shortcuts import render from .models import MyModel # 假设有一个名为 MyModel 的模型 def detail_view(request, pk): obj = MyModel.objects.get(pk=pk) context = {'object': obj} # 将模型实例添加到上下文（context） return render(request, 'my_template.html', context)
  • 在这个例子中，如果用户未登录，login_required_view 视图会调用 redirect 函数将用户重定向至名为 ‘login’ 的URL对应的视图（通常是登录页面）。
  • 如果用户已登录，则继续执行原视图中的其他逻辑。

【三】form表单的下载和上传

【1】form标签属性的方法解析

  • form标签里面的enctype属性

    • enctype对应的有三个方法

  • 方法一：enctype = multipart/form-data

    • 键值对还是会按照原来的方式进行提交，文件对象会被单独处理
    • 数据到了后端提取的时候，键值对数据直接get

  • 方法二：enctype = application/x-www-form-urlencodeda

    • 默认enctype属性的值
    • 默认按照键值对进行处理数据，不会对键值对以外的数据进行额外的处理，比如文件数据，只能传入文件的名字

  • 方法三很少用：enctype = text/plain

    • 表单数据将不会被进行特定的编码或处理，而是直接作为纯文本发送
    • 能会导致解析问题或数据损坏

【2】request.POST的方法

  • 用于接收通过HTTP POST方法发送到服务器的数据
  • 用户通过HTML表单提交数据（通常是通过点击一个<input type="submit">按钮或类似的机制），并且表单的method属性设置为"POST"时，这些数据就会被发送到服务器
  • 可以在后端通过request.POST.get("键")获取值参数
  • get不出来值返回None
```python http://example.com/foo/bar/?page=2

【3】request.path的方法

  • 只能获取到路由地址，无法获取到参数
```python http://example.com/foo/bar/
  • foo/bar/

【5】request.get_full_path()的方法

  • 该方法返回请求URL的完整路径，包括路径部分和任何查询参数

  • 即能获取到路由地址又能获取到完整的路由地址后面的参数

```python http://example.com/foo/bar/?page=2
  • /foo/bar/?page=2

【7】request.method的方法

  • 该方法返回客户端用于发起请求的HTTP方法
  • 返回的是"GET、POST…"请求
  • 定义请求的类型

【8】request.FILES的方法

  • 该属性是一个类似字典的对象，包含了请求中通过文件上传组件发送的所有文件

  • 通过request.FILES访问上传的文件

  • 通过request.FILES.get("键")访问单个值
