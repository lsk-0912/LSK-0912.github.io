Mysql之约束条件

  • 前面已经学了not null和null，unique，zerofill约束条件

【一】什么约束条件

  • 就是创建表格时，限制表中的数据，保证添加到表中的数据准确和可靠性！
    • 不符合就会插入失败
  • 修改表的时候也可以添加约束条件

【二】约束条件概括

  • null ：可以为空

  • not null : 非空约束

    • 不可以为空

  • unique ：唯一性约束

    • 插入的数据不能和表中的数据重复（数据在表中只能出现一次），否则会报错

  • zerofill：默认用0填充至指定长度

  • primary key : 主键约束pk

    • 增加查询效率

  • foreign key ：外键约束fk

    • 表与另一张表之间的联系
  1. 约束条件就是为了限制表中的数据，保证数据的准确性和可靠性而存在
  2. 约束条件在创建表时根据需求可加可不加

【0】如何查看当前表的约束条件/字段限制

  • 给约束条件添加名字
  • 查看当前所有的库

  • 进入到information_schema库中

  • 查看所有的表

  • table_constraints 该表专门存储约束信息

  • 查看某个表的约束条件

【1】null

  • 数据可以为空

【2】not null

  • 数据不能可空，必须给值

【3】unique

  • 保证数据的唯一性，确保表中无相同数据
  • 但是可以为空

【4】主键约束PK（primary key）

  • 主键约束–》不能为空，而且唯一，一张表只能有一个主键约束

（1）主键术语

  • 主键约束–》给表中的某个字段添加主键约束
  • 主键字段–》表中的某个字段添加了主键约束
  • 主键值–》主键字段出现的每个值就是主键值

（2）作用

  • 主键约束不能为空，不能重复出现
  • 'not null，unique’的效果也是不能为空，不能重复出现
  • 二者本质不同，添加主键约束之后还会自动添加‘索引–index’；
  • 一张表没有主键是无效的，主键是当前行的唯一标识。
  • 即使两行数据相同，但是根据主键的不同，两行数据还是不同的

（3）主键根据个数分类

  • 单一主键，复合主键
    • 单一主键：给字段添加一个主键约束
    • 复合主键：给多个字段联合添加一个主键约束
[1]单一主键：列级约束
  • if exists ：有责删除，无责什么都不发生

[2]单一主键：表级约束

  • 给主键重命名

[3]复合主键：只能为表级约束
  • 复合主键，表级约束，并且给其重命名

(4)主键根据性质分类

  • 自然主键：主键是个自然数，这个主键与业务没有任何关系
  • 业务主键：主键值与表中的业务密切相关。这种业务发生改变时，业务主键也会跟着受影响。

(5)自动生成主键

  • 使用字段auto_increment

  • mysql数据库系统提供一个自然自增数据，专门用来生成主键值，主键值不需要用户去维护，也不需要生成，mysql自动生成

  • 默认数字从1开始，从1开始递增

  • 插入数据

  • 每次递增的数字只会出现一次

(6)重置主键起始位置

[1]清空表数据

  • 将原来表的数据全部清空，就重置了主键的起始位置

  • 不建议使用

[2]指定主键的起始位置
  • 使用表修改语句将带有 AUTO_INCREMENT 的字段数字指定为自定义位置
  • 字段数字指定为自定义位置,指定的位置就会继续递增

【5】外键约束FK（foreign key）

（1）引入

  • 某个字段依赖于另一张表中某个字段的值，而被依赖的字段必须具有主键约束或者唯一约束，被依赖的数据表称为主表（父表），设置外键约束的表称为子表或者从表。

  • 上表中

  • 主表就是班级表–》班级号–》主键

  • 外表就是学生表–》班级号–》外键

（2）外键术语

  • 外键约束–》给表中的某个字段添加外键约束
  • 外键字段–》某个字段添加了外键约束
  • 外键值–》外键字段中的每一个数据都是外键值

（3）外键的关系

  • 一对一–》好比：人和人的信息
  • 一对多–》好比：书和印刷厂
  • 多对多–》好比：书和作者

（4）外键语句

  • 先建立需要被建立关系的表，这样才能让建立关系的表去和它建立关系
  • 建立一个建立关系的表
  • 和它建立关系的表

（5）一对多外键关系

  • 一对多：外键关系应该健在多的一方
    • 就好比：部门只有一个，但是部门有好多个员工
  • 部门表先创建

  • 有了部门表在创建员工表，并且要员工和部门绑定

  • 插入数据

  • 级联更新on update cascade，级联删除on delete cascade

（6）外键字段的约束效果

  • 建表时应该先创建被关联的表（没有外键字段的表）
  • 插入数据的时候 应该先插入被关联表(没有外键字段的表)
    • 外键字段填入的值只能是被关联表中已经存在的值
  • 修改、删除被关联表数据都会出现障碍

（7）多对多关系

  • 好比书和作者

    • 书有好几个作者
    • 作者发出好多本书

  • 多对多无法判断谁先创建谁后创建

  • 需要借助第三张表专门建立关系

  • 建立第三张表来储存它们的关系

  • 两张表内的数据没有在第三张表内绑定关系的情况下随意新增修改和删除

（8）一对一关系

  • 一对一关系外键字段在任意一方都可以，推荐在查询率较高的一方

【补充】

  • 外键根据个数分类

    • 单一外键【给一个字段添加外键约束】
    • 复合外键【给多个字段联合添加成一个外键约束】

  • 一张表可以有多个外键字段

  • 外键字段可以为null

  • 外键字段引用一张表的某个字段时，被引用的字段必须具有unique约束

  • 有了外键引用之后，表分为父表和子表。一对多关系中，一为父表，多为子表。创建表时，先创建父表，再创建子表；插入数据时先插入父表中的数据，再插入子表中的数据；删除数据时，需先删除子表的数据，再删除父表的数据

  • 外键约束没有列级约束，只有标记约束

【三】级联更新和级联删除

  • 什么是级联更新
    • 当主表中的某条记录的值发生改变时，从表中的相应记录的值也会自动发生相应的变化，保持数据的一致性。
  • 什么是级联删除
    • 当主表中的某条记录被删除时，从表中的相应记录也会自动被删除，确保了当主关键字段中相应的记录被删除时，会自动删除相关表中对应的记录，从而保持数据的一致性和完整性。

【1】介绍

  • 添加级联更新和级联删除时需要在外键约束后面添加
  • 在删除父表中的数据时，级联删除子表中的数据 on delete cascade
  • 在更新父表中的数据时，级联更新子表中的数据 on update cascade
  • 以上的级联更新和级联删除谨慎使用，因为级联操作会将数据改变或删除【数据无价】
  • 在修改约束条件时，建议可以将原先的约束删除再重新添加约束条件
