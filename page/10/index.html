4.2生成器

发表于 2025-10-15 分类于 Blog

【一】生成器

【1】什么是生成器

  • 在py中生成器是一种特殊的迭代器，可以在需要时生成数据，不必提前从内存中生成并储存整个数据集
  • 通过生成器，可以逐个生成序列中的元素，而无需一次性生成整个序列
  • 生成器在处理大数据时，具有节约内存，提高效率的特点

【2】生成器的创建方式

【1】生成器的表达式

  • 使用列表推到式时，可以将列表推导式的方括号改为圆括号，即可创建一个生成器
```python #推导式生成列表 data_list = [i for i in range(6)] print(data_list)#[0, 1, 2, 3, 4, 5] #将列表改成元祖，看起来像元祖推导式，其实是一个生成器对象 data_list = (i for i in range(6)) # 这个可不是元组，是生成器的表达式 print(data_list)# at 0x000001D8A3A48350> #生成器对象可以强转成列表 print(list(data_list))#[0, 1, 2, 3, 4, 5]

【2】yield关键字

  • 使用yield关键字定义一个生成器函数时，生成器函数中的yield语句会暂停函数执行并返回一个值，下一次调用该函数时会继续执行并返回下一个值。
```python def add(i): yield i # add(6) 会进入到add函数将原本的函数对象转换为生成器对象 res = add(6) # # 想取值只能是 next 迭代取值 print(res.__next__()) ``````

hashlib模块

  • 摘要算法：只能加密不能解密
  • 加密算法：用方法加密，加密后的字符串可以解密

【一】摘要算法

  • Python的hashlib提供了常见的摘要算法

    • 如MD5,SHA1

  • 摘要算法又称哈希算法、散列算法。

  • 它通过一个函数，把任意长度的数据转换为一个长度固定的数据串（通常用16进制的字符串表示）。

  • 摘要算法就是通过摘要函数f()对任意长度的数据data计算出固定长度的摘要digest

  • 目的是为了发现原始数据是否被人篡改过。

  • 摘要算法之所以能指出数据是否被篡改过，就是因为摘要函数是一个单向函数

  • 计算f(data)很容易，但通过digest反推data却非常困难。

  • 而且，对原始数据做一个bit的修改，都会导致计算出的摘要完全不同。

  • md5只能对二进制数据进行加密

【三】加盐

  • 如果以明文保存用户口令，如果数据库泄露，所有用户的口令就落入黑客的手里。
  • 此外，网站运维人员是可以访问数据库的，也就是能获取到所有用户的口令。
  • 正确的保存口令的方式是不存储用户的明文口令，而是存储用户口令的摘要
  • 由于常用口令的MD5值很容易被计算出来
    • 所以，要确保存储的用户口令不是那些已经被计算出来的常用口令的MD5
    • 这一方法通过对原始口令加一个复杂字符串来实现，俗称“加盐”

【一】函数

  • 什么是函数

      • def name_age(name,age)#调用的时候别忘记给参数
    • 表达式调用
      • data = age(2,3) + 6#age()是函数
    • 函数作返回值
      • 函数的返回值作为参数，给其他参数使用
      • 函数的内存地址可以作为参数

  • 函数的形参

    • 就是在定义函数（）里面的参数名

  • 函数的实参

    • 就是函数被调用阶段进去的参数

  • 传参

    • 关键字传参
    • 默认值传参

  • 可变长位置擦参数

  • 可变长关键字参数

    • **kwargs

  • 全局名称空间：自己定义的

  • 局部名称空间：定义在函数/类 里面的函数名…

  • 程序启动的加载顺序–

    • 先内-再全-最后局

  • 寻找变量名的加载顺序

    • 先局-在全-最后内

【三】作用域

  • 局部作用域（Local）

  • 查找顺序legb

  • 函数可以直接被引用
    • 定义一个函数用一个新的变量来存，用新的变量名调用
  • 可以做为元组被储存
  • 函数可以作为一个参数传给另一个参数
  • 函数的返回值可以是一个参数传给另一个参数
  • 函数的返回值可以是一个函数
    • 直接将函数的内存地址返回
