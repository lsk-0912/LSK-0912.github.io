os模块

【一】什么是os模块

  • os模块可以操作本地的文件夹及文件

  • 创建删除文件夹/文件

  • 执行本地的cmd命令

  • 拼接路径

  • __file__表示当前文件

【二】文件路径相关操作(path)

【1】获取当前绝对路径（abspath）

【2】获得当前文件所在文件路径(dirname)

【3】判断当前文件路径是否存在----》文件夹/文件(exists)

【4】拼接文件路径（join）

【5】切割文件路径

  • 切后得到一个元组里面有两个元素，两个元素分别是（其他路径，最后一个文件夹/文件夹名）

【6】获取文件结尾的文件/文件夹名（basename）

【7】判断当前文件是不是文件夹/文件（isdir,isfile）

  • 判断是否是文件isfile
  • 判断是否是文件夹isdir

【8】判断当前路径是否是绝对路径(isabs)

【9】获取当前文件或目录的最后访问时间（getatime）

【10】获取当前文件或目录的最后修改时间（getmtime）

【11】获取当前文件或者目录的创建时间(getctime)

【12】获取当前文件的大小（getsize）

【三】文件和文件夹操作

【1】创建文件夹（mkdir）

【2】创建多级文件夹(makedirs)

【3】删除单级文件夹(rmdir)

  • 只能删除空的文件夹 即使文件夹内有空文件夹也不算空的

【4】删除多级文件夹(removedirs)

  • 删除多级文件夹目录 路径一定是到最里面那一层并且每个文件夹必须是空的

【5】列出当前文件夹下所有文件名(listdir)

【6】删除指定文件(remove)

【7】给指定文件夹和文件重命名（rename）

  • 给两个绝对路径
    • 一个是之前的绝对路径
    • 一个是重命名后的绝对路径

【8】获取当前文件/文件夹详细信息，元信息（stat）

【9】获取当前工作目录(getcwd)

【10】切换工作(chdir)

【补充】执行shell命令

  • 执行cmd命令

【补充】获取当前系统所对行的默认分隔符

【记住重点】

  • os.path.join() 拼接路径
  • os.path.dirname( __file__ ) 获取当前文件所在的文件夹路径
  • os.path.abspath(__file__ )获取当前文件所在的文件路径
  • os.path.exists() 判断路径是否存在
  • os.mkdir() 创建单级文件夹
  • os.makedirs() 创建多级文件夹
  • os.makedirs(path, exist_ok=True) 自动创建不存在的多级文件夹路径

4.21大总结

【一】函数

  • 什么是函数

    • 就是把一个复杂的需求拆分，然后做成单个功能的，在全局都能用（前提是在调用函数的下面使用）

  • 函数的标志-----》 def打头

  • 函数的调用方式

    • 直接掉调用
      • def name_age(name,age)#调用的时候别忘记给参数
    • 表达式调用
      • data = age(2,3) + 6#age()是函数
    • 函数作返回值
      • 函数的返回值作为参数，给其他参数使用
      • 函数的内存地址可以作为参数

  • 函数的形参

    • 就是在定义函数（）里面的参数名

  • 函数的实参

    • 就是函数被调用阶段进去的参数

  • 传参

    • 位置传参
    • 关键字传参
    • 默认值传参

  • 可变长位置擦参数

    • *ages

  • 可变长关键字参数

    • **kwargs

【二】名称空间

  • 内建名称空间：python自带的

  • 全局名称空间：自己定义的

  • 局部名称空间：定义在函数/类 里面的函数名…

  • 程序启动的加载顺序–

    • 先内-再全-最后局

  • 寻找变量名的加载顺序

    • 先局-在全-最后内

【三】作用域

  • 局部作用域（Local）

    • 函数亡，局部变量亡
    • 范围：整个函数体范围
    • 查看局部变量：locals

  • 嵌套作用域(Enclosed)

    • 范围：闭包函数

  • 全局作用域（Global）

    • 在模块文件最顶层生命的变量，具有全局性
    • 修改全局变量我们可以用globals

  • 内建作用域（Built）

    • 随着python启动自己加载的名称空间

  • 查找顺序legb

  • 全局变量和局部变量重名时，采用就近原则

【四】函数对象

  • python中一切皆对象
  • 函数可以直接被引用
    • 定义一个函数用一个新的变量来存，用新的变量名调用
  • 可以做为元组被储存
  • 函数可以作为一个参数传给另一个参数
  • 函数的返回值可以是一个参数传给另一个参数
  • 函数的返回值可以是一个函数
    • 直接将函数的内存地址返回

【五】装饰器

  • 在不修改被装饰对象源代码和调用方式的前提下为被装饰对象添加额外的功能。
  • 分类
    • 有参函数
    • 无参函数
  • 原理
    • 闭包函数和函数对象的组合使用
  • 无参装饰器
    • 没有参数的装饰器
  • 模板

  • 有参装饰器

    • 根据指定的位置传递参数

  • 优化

    • 用可变长位置参数和可变长关键字参数接收函数所需要的所有参数

  • 模板

  • 装饰器语法糖

  • 标志@

  • 无参装饰器语法糖

    • 无参数

  • 有参语法糖

    • 有参数
    • 但语法糖：就@
    • 多语法糖：多个@

【六】迭代器

  • 每次迭代得到的返回值结果可就是下一次迭代开始的初始值
  • 可迭代对象
    • 内置有.__iter__方法的对象都是可迭代对象
    • 不可迭代的：布尔，整数，浮点数，剩下都的可以

【七】迭代器对象

  • 内置有.__iter__和.__next__就是迭代器对象

  • .__next__挨个取值一次只能取一个，下次从你上一次的取值结果继续开始

  • 也可以直接写成next（），iter（）

  • 缺点：除非取值取到尽头，否则永远不知道终止索引在哪里

    • 只能取一次记一次位置，下一次的起始位置是上一次的终止位置，想回到开头，回不去

【八】生成器

  • 在python中生成器就是一种特殊的迭代器

  • 创建方式

    • 可以将列表推导式的方括号改为圆括号，即可创建一个生成器

    • yield关键字

      • 生成器函数中的yield语句会暂停函数执行并返回一个值，下一次调用该函数时会继续执行并返回下一个值
      • send（）传值

  • 模块

  • from 模块名 import 属性名

  • import 模块名

  • 支持导入多个模块

  • 重命名模块—》import 模块名 as 新模块名

  • 模块分类

    • 纯python编写的自己定义的模块
    • 内置的一系列模块 (python解释器自带的和你安装 pip install )
    • 使用c语言编写的底层代码
    • c和c++扩展出来的模块

  • 加载顺序

    • 跟上面反过来

  • 查找顺序

    • 跟加载顺序相反

  • 包

    • 模块的聚集地
    • 里面有.__init__文件的就是包

  • from 模块名 import 属性名

  • from 包名 import 模块名

【十】绝对路径

  • 在python查找的顺序就是绝对路径

  • 你所在的文件夹就是相对历经
  • ./当前同目录下载
  • .//当前目录上一级目录下载
  • 默认就是当前目录下载

【十二】json模块

  • json是 你存进去是什么类型读出来就是什么类型

  • 导入模块

    • import json模块

  • 保存为json文件

    • with open…

  • 读出来转为python对象

    • with open…

【十二】time模块

  • 生成时间戳：time.time()

  • time.sleep():睡眠时间

  • 生成时间戳转为时间元组

    • 国际时间time.gmtime(time.time)
    • 本地时间time.localtime(time.time)

【2】回想一下wsgiref模块 的请求周期

  • 前端发起请求 —>
  • wsgiref 模块的 Request 上 —>
  • 获取 路径 —
  • urls.py 中映射视图函数—
  • 进入到视图函数中 —
  • 读取前端模版 / 读取数据库数据 ----
  • 经过 wsgiref 模块的 response 对象封装—
  • 交给前端页面 渲染数据

【3】Django的请求生命周期

  1. 从前端入手 前端发送请求
  2. wsgiref 模块上对 Request 进行分装
  3. Django框架上 中间件 （相当于你的公司的保安 你的第一道防线）
  4. Django框架上的 路由映射 urls.py 去根据路径映射制定的视图函数
  5. Django框架上的 视图函数 views.py 中
    • 获取前端页面 (模版语法渲染)
    • 读取数据库数据 (增删改查)
    • 记录你的操作日志 (记录日志)
    • 返回 response对象
  6. Django框架上的 urls.py 根据制定路径回去
  7. 回到了 wsgiref 模块上 对Django的 response 对象进行封装
  8. 回到前端解包渲染

面向对象

【一】简介----》引入

  • 将数据和功能进行绑定
  • 不再是所有数据都可以调用任意的功能
  • 上述将数据与功能整合到
  • 可以理解为：可以理解我们写的面条版程序，一步接着做

【二】面向对象和 面向过程

【1】面向过程

  • 核心在于"过程"
    • 将程序"流程化"
  • 过程就像流水线，用来分步骤解决问题
  • 过程指是解决问题的步骤，先干什么在干什么
  • 面向对象的设计就好比精心设计好一条流水线，是一种机械式的思维方式。

【2】优点，缺点

  • 优点

    • 复杂度的问题流程化，进而简单化（一个复杂的问题，分成一个个小步骤，去实现晓得步骤就会相对简单）

  • 缺点

    • 一套流水线或者流程就是用来解决一个问题。改一个组件，牵一发而动全身。

【3】面向对象

  • 核心在于"对象"二字。

  • 对象的终级奥秘就是将程序"整合"。

  • 对象就是"容器"，用来存放数据与功能

  • 文字版理解

    • 面向对象就相当于上帝，在上帝的视角，人是对象，动物是对象，石头是对象，水是对象，山是对象…存在的可以扩充，不存在的可以创造。

  • 可以这样理解面向对象就像我们要设计的校园模拟器，需要在校园中创造各种生活场景。

    • 一个学校：下面有学生，老师，运动员等。。。
    • 而每一个角色都会对应其具有的责任和功能
      • 比如一个学生具有姓名、年龄、性别、所在班级等。
      • 学生还有能做的活动，读书、写作、跑步、打篮球等。
      • 比如一个老师可以教书、批改作业等。
      • 比如运行员可以参加训练和比赛等。

  • 可以理解为学校就是一个对象

【4】优缺点

  • 优点
    • 解决了程序的扩展性
    • 对某一个对象单独修改，会立刻反映到整个体系中，比如对游戏中的人物和场景进行修改就会很容易，而且是立刻有成效。
  • 缺点
    • 编程的复杂度远高于面向过程，不了解面向对象而立即上手基于它设计程序，极容易出现过度设计的问题。
    • 无法向面向过程的程序设计流水线式的可以很精准的预测问题的处理流程与结果，面向对象的程序一旦开始就由对象之间的交互解决问题，即便是上帝也无法准确地预测最终结果。

【五】菱形继承-----》继承实现的原理

  • 菱形继承问题，一个类可以继承多个父类，父类又可以继承父类

  • 当类是经典类时，多继承情况下，在查找属性不存在时，会按照深度优先的方式查找下去

【2】广度优先

  • 发生在新式类上，在当前内部找不到指定属性的收会想上找

  • 查找顺序就是广度优先

  • 当类是新式类时，多继承情况下，在查找属性不存在时，会按照广度优先的方式查找下去

派生

【一】什么是派生

  • 派生就是子类继承父类所有的属性和方法，并且派生出自己的属性与方法

【1】子类继承父类的属性

【2】继承方法一

  • 用的时候直接用类名调用函数方法
    • People.__init__(self, name, age, sex)
    • 直接调用就可以用这个函数的属性和方法

【3】继承方法二

  • 调用**super()**会得到一个特殊的对象
  • 该对象专门用来引用父类的属性
  • 且严格按照MRO规定的顺序向后查找
  • MRO:查看继承的顺序
  • 优缺点：super()，只会拿类名（）里面的第一个父类的属性和方法

【4】总结

  • 这两种方式的区别是：
    • 方式一是跟继承没有关系的，而方式二的super()是依赖于继承的
    • 并且即使没有直接继承关系，super()仍然会按照MRO继续往后查找

【5】组合

  • 在一个类中以另外一个类的对象作为数据属性，称为类的组合。
  • 有所有父类的属性和方法
  • 此时对象teacher1集对象独有的属性、Teacher类中的内容、Course类中的内容于一身（都可以访问到），是一个高度整合的产物
  • 当类之间有显著不同，并且较小的类是较大的类所需要的组件时，用组合比较好

【6】组合和继承的区别

  • 组合与继承都是有效地利用已有类的资源的重要方式。但是二者的概念和使用场景皆不同
  • 继承的方式
    • 通过继承建立了派生类与基类之间的关系，它是一种’是’的关系，比如白马是马，人是动物。
    • 当类之间有很多相同的功能，提取这些共同的功能做成基类，用继承比较好，比如老师是人，学生是人
  • 组合的方式
    • 用组合的方式建立了类与组合的类之间的关系，它是一种‘有’的关系,比如教授有生日，教授教python和linux课程，教授有学生s1、s2、s3…
